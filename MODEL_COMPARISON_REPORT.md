## 推荐配置

### 场景 1: 追求 NSFW 零漏报（成人内容过滤）
- **模型**: 优化模型 (prod_30epoch)
- **分辨率**: 160²
- **优势**: NSFW Recall 100%，速度最快
- **代价**: 2 个 SFW 误报（可接受）

### 场景 2: 追求 SFW 零误报（正常内容保护）
- **模型**: GitHub 原始模型
- **分辨率**: 320²
- **优势**: SFW Accuracy 100%
- **代价**: 2 个 NSFW 漏报（危险）

### 场景 3: 平衡方案
- **策略**: 两阶段检测
  1. 优化模型快速初筛 (160², 1.64s)
  2. 对阳性样本用原始模型复核 (320², 3.62s)
- **总耗时**: 1.64s (阴性) 或 5.26s (阳性需复核)
- **效果**: NSFW Recall 100% + SFW 误报降低

## 性能优化效果

| 优化项 | 提升 |
|--------|------|
| mamba2 CUDA kernel | 训练稳定 |
| motion_gate | 精度持平 |
| sparse_detect | 推理加速 |
| channels_last | 内存优化 |
| BF16 混合精度 | 速度 +20% |
| 分辨率 320→160 | **速度 +2.2×** |

**综合加速**: 3.62s → 1.64s (**+2.2× 速度**)

## 结论

1. **优化模型 (160²) 推荐用于生产**
   - NSFW Recall 100% (零漏报)
   - 速度最快 (1.64s/video)
   - 87.5% 总精度与原模型持平

2. **GitHub 原始模型不推荐**
   - NSFW 漏报 33%，无法接受
   - 速度慢 2.2×

3. **A10 高分辨率模型失败**
   - 512² 训练过拟合
   - 精度反而下降到 75%

4. **mamba3 在 A10 24GB 无法运行**
   - 所有分辨率 OOM
   - 需要 A100 80GB
